# CocoaBoard (R Shiny)

Chocolate Sales Dashboard — R/Shiny port of the visualization components from [DSCI-532_2026_8_cocoaboard](https://github.com/UBC-MDS/DSCI-532_2026_8_cocoaboard). No AI chat tab; dashboard only (filters, KPIs, map, leaderboard, revenue trend).

**Deployed app:** [Add your Posit Connect URL here after deployment, and add the same URL to the repository **About** section (Settings → About → Website).]

## Features

- **Filters:** Date range, Country (multi), Product (multi)
- **Value boxes:** Total Revenue, Total Boxes Shipped, Active Sales Reps, Avg Revenue, YoY Revenue, MoM Revenue (with loading spinners)
- **Revenue by Product:** Pie chart (filtered by date, country, product)
- **Leaderboard table:** Rank, Revenue, Transactions, Boxes, Avg Deal, Rev Share + summary row
- **Revenue trend:** Line chart for top 5 sales reps by month

## Data

- **Source:** `data/raw/Chocolate_Sales.csv` (same as the Python CocoaBoard project).
- Ensure the file exists at `data/raw/Chocolate_Sales.csv` relative to the project root.

## Run locally

```r
# Install dependencies (once)
install.packages(c("shiny", "bslib", "ggplot2", "dplyr", "tidyr", "plotly", "DT", "lubridate", "bsicons", "scales", "shinycssloaders"))

# From project root (cocoaboard-visualisation-R/)
shiny::runApp()
```

Or in RStudio: open `cocoaboard.Rproj`, then run the app (e.g. Run App on `app.R`).

## Deploy to Posit Connect Cloud

Connect Cloud needs a **generated** `manifest.json` (from `rsconnect::writeManifest()`), not a hand-written one. Errors like *"Download of the ggplot2 package has failed"* or *"Package X has invalid package source"* mean the manifest was not generated by rsconnect.

### Step 1: Generate `manifest.json` (do this before first deploy)

From your machine, with the app's R packages installed, run **one** of these from the **project root** (folder that contains `app.R`):

**Option A – R script (recommended):**
```bash
cd /path/to/cocoaboard-visualisation-R
Rscript scripts/write_manifest.R
```

**Option B – in R:**
```r
setwd("/path/to/cocoaboard-visualisation-R")
install.packages("rsconnect")
rsconnect::writeManifest(appDir = ".", appPrimaryDoc = "app.R", appMode = "shiny")
```

Then **commit and push** the updated `manifest.json`. It will include all R dependencies and the metadata Connect needs to install them.

### Step 2: Deploy on Connect

1. Push the repo (including the generated `manifest.json`) to GitHub (or your Git host).
2. In [Posit Cloud](https://posit.cloud/), open Connect → new content → **Shiny Application**.
3. Connect the repo; leave **Subdirectory** blank; set **Application path** to `app.R` if needed.
4. Deploy. Connect uses only `manifest.json` for R packages (no renv).
